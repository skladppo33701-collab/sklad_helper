# i18n Guidelines

Supported locales:
- ru (default)
- kk
- en

Rules:
- All UI strings go through localization files.
- Product names and transfer names from Excel/1C are NOT translated.
- Status labels are localized.
- Error messages (scan errors, validation errors) must be localized.

Settings:
- user locale stored in `users/{uid}.locale`
- app reads locale on startup and applies.

Fallback:
- if translation missing -> ru fallback
